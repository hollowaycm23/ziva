PROBLEMAS COMUNS E SOLUÇÕES
===========================

❌ "service "ollama-server" not found"
→ Você está usando docker-compose.yml OLD
→ Solução: Atualizar para nova versão

❌ "network "ziva-net" not found"
→ Network não foi criada
→ Solução: docker network create ziva-net

❌ "connection refused" na porta 11434
→ Ollama não inicializou ainda
→ Solução: Aguardar 30s adicional

❌ "CrashLoopBackOff" em ziva-core
→ Ver logs: docker logs ziva-core
→ Problemas comuns:
   - ZIVA_LLM_BASE_URL incorreta
   - Falta de QDRANT_URL
   - Diretórios /app/data, /app/logs sem permissão

SCRIPTS ÚTEIS
=============

Diagnosticar:
  bash scripts/check_docker_config.sh

Deploy com teste:
  bash scripts/deploy_and_test.sh

Ver logs:
  docker logs -f ziva-core
  docker-compose logs -f

Limpeza completa:
  docker-compose down -v
  docker network rm ziva-net 2>/dev/null || true
  rm -rf ./qdrant_storage ./logs ./data ./db ./inbox ./outbox

Reiniciar específico:
  docker restart ziva-core
  docker-compose restart
