🚀 COMO COMEÇAR - PASSO A PASSO
================================================================================

PASSO 1: Verificar Pré-requisitos (5 minutos)
   • Docker 20.10+: docker --version
   • Docker Compose 2.0+: docker-compose --version
   • RAM: 16GB+ (verificar com free -h ou wmic)
   • Espaço: 50GB+ disponível (verificar com df -h)

PASSO 2: Criar Network (1 minuto)
   docker network create ziva-net 2>/dev/null || true

PASSO 3: Gerar Secrets (2 minutos)
   bash scripts/init_env.sh
   Editar .env com valores personalizados se desejado

PASSO 4: Validar Configuração (5 minutos)
   bash scripts/check_docker_config.sh
   Verificar output (todas as checagens devem passar)

PASSO 5: Build Docker (5-10 minutos)
   docker build -t ziva:latest .
   Aguardar conclusão (multi-stage build: Go + Python)

PASSO 6: Deploy (1 minuto)
   docker-compose up -d
   9 containers serão iniciados

PASSO 7: Aguardar Inicialização (60 segundos)
   sleep 60

PASSO 8: Verificar Saúde (2 minutos)
   docker ps --filter "name=ziva"
   curl http://localhost:8000/v1/health
   curl http://localhost:11434/api/tags
   curl http://localhost:6333/health

PASSO 9: Acessar Interface (1 minuto)
   Open WebUI: http://localhost:3000
   API: http://localhost:8000
   Ollama: http://localhost:11434

PASSO 10: Monitorar Logs (contínuo)
   docker logs -f ziva-core

TEMPO TOTAL: ~30-40 minutos

🔧 TROUBLESHOOTING RÁPIDO
================================================================================

ERRO: "service "ollama-server" not found"
→ SOLUÇÃO: Você ainda tem docker-compose.yml antigo
  docker-compose down
  docker-compose up -d

ERRO: "network "ziva-net" not found"
→ SOLUÇÃO: Network não foi criada
  docker network create ziva-net
  docker-compose up -d

ERRO: "connection refused" na porta 11434
→ SOLUÇÃO: Ollama não inicializou ainda (normal)
  Aguardar 30 segundos adicionais
  docker logs ziva-ollama

ERRO: "CrashLoopBackOff" em ziva-core
→ SOLUÇÃO: Ver logs
  docker logs ziva-core
  Problemas comuns:
    - ZIVA_LLM_BASE_URL incorreta em .env
    - Falta de espaço em disco
    - Permissões em /app/data

ERRO: "port 8000 already in use"
→ SOLUÇÃO: Outra aplicação usando a porta
  docker-compose down
  Liberar porta 8000 (lsof -i :8000)
  docker-compose up -d
